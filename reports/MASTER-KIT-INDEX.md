# Мастер-комплект услуги: юридическая проверка квартиры перед покупкой

Этот комплект собирает в одном месте все материалы по услуге: от шаблонов отчёта до упаковки, витрины и сценария запуска.

---

## 1. Что входит в мастер-комплект

### 1. Полный шаблон отчёта
**Файл:** `reports/apartment-legal-check-template.md`

Когда использовать:
- когда нужен полный каркас отчёта;
- когда проверка идёт глубоко;
- когда нужно не забыть ни один блок риска.

Что внутри:
- структура полного отчёта;
- источники данных;
- разделы по праву, обременениям, продавцу, спецрискам;
- матрица рисков;
- итоговое заключение.

---

### 2. Боевая версия отчёта
**Файл:** `reports/apartment-legal-check-operational-template.md`

Когда использовать:
- в реальной ежедневной работе;
- для быстрой сборки заключения по объекту;
- для экспресс и стандартной проверки.

Что внутри:
- карточка объекта;
- быстрый вердикт;
- блок «что уже проверено»;
- красные флаги;
- спецриски;
- финальная рекомендация.

---

### 3. Регламент услуги
**Файл:** `reports/apartment-legal-check-regulation.md`

Когда использовать:
- чтобы понимать порядок работы;
- чтобы не путаться в этапах;
- чтобы одинаково вести все проверки.

Что внутри:
- форматы услуги;
- этапы работы;
- источники проверки;
- правила выводов;
- стоп-факторы;
- стандарт качества.

---

### 4. Упаковка услуги
**Файл:** `reports/apartment-legal-check-packaging.md`

Когда использовать:
- для описания услуги клиенту;
- для офферов;
- для пакетов и объяснения ценности.

Что внутри:
- название услуги;
- описание;
- пакеты;
- что входит / не входит;
- что можно обещать;
- тексты для общения;
- ответы на возражения.

---

### 5. Витрина услуги
**Файл:** `reports/apartment-legal-check-showcase.md`

Когда использовать:
- для Telegram-поста;
- для закрепа;
- для сайта или карточки услуги;
- для коротких продающих текстов.

Что внутри:
- посты;
- офферы;
- FAQ;
- блоки «кому подходит», «что проверяется», «что получает клиент».

---

### 6. Комплект под запуск
**Файл:** `reports/apartment-legal-check-launch-kit.md`

Когда использовать:
- чтобы реально стартовать услугу;
- для переписки с клиентами;
- для квалификации обращений;
- для выдачи результата.

Что внутри:
- пакеты и ориентиры по прайсу;
- анкета на вход;
- сценарий переписки;
- шаблон финальной выдачи;
- офферы;
- FAQ;
- логика перевода интереса в заказ.

## 2. Карта процесса: от заявки до результата

### Шаг 1. Получить входные данные
Минимум:
- адрес;
- кадастровый номер, если есть;
- ссылка на объявление;
- цена;
- кто продаёт;
- что уже есть из документов.

Для этого использовать:
- `apartment-legal-check-launch-kit.md`
- `apartment-legal-check-packaging.md`

### Шаг 2. Определить формат проверки
- **Экспресс** — быстрый отсев;
- **Стандарт** — перед авансом;
- **Расширенная** — сложный объект / сложная история.

Для этого использовать:
- `apartment-legal-check-regulation.md`
- `apartment-legal-check-launch-kit.md`

### Шаг 3. Провести проверку
- идентифицировать объект;
- проверить право;
- проверить обременения;
- проверить историю переходов;
- проверить продавца;
- оценить спецриски.

Для этого использовать:
- `apartment-legal-check-operational-template.md`
- `apartment-legal-check-template.md`

### Шаг 4. Сформулировать вывод
- уровень риска;
- главные красные флаги;
- чего не хватает;
- можно ли идти дальше;
- при каких условиях допустима сделка.

Для этого использовать:
- `apartment-legal-check-operational-template.md`
- `apartment-legal-check-regulation.md`

### Шаг 5. Выдать результат клиенту
- короткая версия;
- полная версия;
- рекомендации по следующим шагам.

Для этого использовать:
- `apartment-legal-check-launch-kit.md`
- `apartment-legal-check-showcase.md`

## 6. Быстрый summary всей системы

### Что уже готово
- полный шаблон отчёта;
- короткий боевой шаблон;
- регламент проведения проверки;
- упаковка услуги;
- витрина для продвижения;
- комплект для запуска.

### Что это даёт
Сейчас услуга уже подготовлена сразу на трёх уровнях:

#### 1. Производство
Можно реально делать проверки и выдавать результат.

#### 2. Продажа
Можно объяснять услугу, публиковать её и отвечать на возражения.

#### 3. Запуск
Можно принимать заявки, квалифицировать клиентов и вести их по понятному сценарию.

---

## 7. Главный принцип мастер-комплекта
Этот комплект строится вокруг одной простой идеи:

**Сначала понять риски сделки, потом передавать деньги.**

Не вокруг пустых обещаний, не вокруг юридического тумана, а вокруг понятной и полезной работы.
