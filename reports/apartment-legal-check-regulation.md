# Регламент услуги: юридическая проверка квартиры

## 1. Цель услуги
Услуга нужна для того, чтобы до аванса, задатка или сделки понять, насколько безопасно покупать конкретную квартиру, какие есть юридические риски, чего не хватает для нормального выхода на сделку и какие условия обязательно нужно включить в процесс покупки.

Результат услуги — не просто «всё ок / не ок», а:
- структурированный отчёт;
- список выявленных рисков;
- перечень недостающих документов;
- рекомендации по безопасному сценарию сделки.

---

## 2. Форматы услуги

### 2.1. Экспресс-проверка
Подходит, если нужно быстро понять, стоит ли вообще продолжать общение по объекту.

**Что входит:**
- базовая идентификация объекта;
- первичная проверка права;
- проверка обременений;
- базовая проверка продавца по открытым источникам;
- предварительная оценка красных флагов.

**Результат:**
- короткий вывод;
- уровень риска;
- список того, что запросить у продавца.

**Когда использовать:**
- до просмотра;
- до аванса;
- при первичном отборе объекта.

### 2.2. Стандартная проверка
Подходит, если квартира всерьёз рассматривается к покупке.

**Что входит:**
- всё из экспресс-проверки;
- анализ истории переходов права;
- расширенная проверка продавца;
- анализ специальных рисков;
- проверка базового пакета документов.

**Результат:**
- полноценный рабочий отчёт;
- список рисков по степени серьёзности;
- условия, при которых сделка допустима.

**Когда использовать:**
- перед авансом;
- перед задатком;
- перед подготовкой договора.

### 2.3. Расширенная проверка
Подходит для сложных объектов, нестандартной истории или когда нужна максимальная осторожность.

**Что входит:**
- всё из стандартной проверки;
- анализ сложных оснований права;
- углублённая оценка рисков наследства / приватизации / маткапитала / доверенности;
- проверка дополнительных документов;
- рекомендации по формулировкам договора и порядку расчётов.

**Результат:**
- финальное заключение по сделке;
- список обязательных условий сделки;
- список стоп-факторов, при которых лучше отказаться.

**Когда использовать:**
- объект с историей наследства;
- дарение, приватизация, доверенность;
- несовершеннолетние;
- признаки спора, долгов, банкротства;
- сомнительное поведение продавца.

---

## 3. Что клиент должен прислать на входе

## Минимум для старта
- адрес квартиры;
- кадастровый номер, если есть;
- ссылку на объявление;
- цену;
- ФИО продавца или представителя, если известны;
- краткое описание ситуации: вторичка, ипотека, альтернатива, срочная продажа и т.д.

## Для стандартной и расширенной проверки
- актуальную выписку ЕГРН;
- выписку о переходе прав;
- правоустанавливающий документ;
- паспорт продавца (или представителя);
- доверенность, если действует представитель;
- согласие супруга / документы по брачному статусу;
- справку о зарегистрированных лицах;
- справку об отсутствии долгов по ЖКУ;
- при необходимости: документы по наследству, маткапиталу, опеке, перепланировке.

---

## 4. Этапы работы

### Этап 1. Приём запроса
На этом этапе нужно:
- зафиксировать объект;
- понять, что именно уже есть;
- определить формат проверки: экспресс / стандарт / расширенная.

### Этап 2. Первичная оценка объекта
Проверяется:
- идентификация объекта;
- совпадение адреса и кадастра;
- базовые сведения о праве;
- обременения и ограничения.

### Этап 3. Проверка истории и продавца
Проверяется:
- цепочка переходов права;
- частота перепродаж;
- основания возникновения права;
- продавец по ФССП, банкротству, судам;
- полномочия представителя.

### Этап 4. Проверка специальных рисков
Проверяется:
- супруг / совместная собственность;
- маткапитал;
- несовершеннолетние;
- наследство;
- приватизация;
- дарение / рента / пожизненное содержание;
- перепланировка;
- зарегистрированные лица.

### Этап 5. Сводка рисков
По итогам проверки каждый риск относится к одной из категорий:
- не выявлен;
- требует документа;
- умеренный риск;
- высокий риск;
- стоп-фактор.

### Этап 6. Финальное заключение
На выходе клиент получает:
- короткий итог;
- подробный отчёт;
- список обязательных документов;
- рекомендации по безопасной сделке;
- прямой вывод: идти дальше или нет.

---

## 5. Источники проверки

### Основные
- ЕГРН / Росреестр;
- выписка о переходе прав;
- документы продавца;
- ФССП;
- ЕФРСБ / Федресурс;
- суды общей юрисдикции / арбитраж;
- реестр нотариальных доверенностей.

### Дополнительные
- документы управляющей компании;
- документы по перепланировке;
- документы по опеке;
- документы о брачном статусе;
- наследственные документы.

---

## 6. Что именно проверяется

### По объекту
- существует ли объект как самостоятельный объект права;
- совпадают ли характеристики;
- нет ли противоречий в описании.

### По праву
- кто текущий собственник;
- на каком основании возникло право;
- когда зарегистрировано право;
- есть ли доли и совместная собственность.

### По ограничениям
- ипотека;
- арест;
- запрет регистрационных действий;
- иные зарегистрированные ограничения.

### По истории
- как часто менялся собственник;
- насколько логична цепочка переходов;
- есть ли подозрительные переходы.

### По продавцу
- долги и взыскания;
- банкротство;
- суды;
- полномочия;
- доверенность.

### По специальным рискам
- супруг;
- маткапитал;
- несовершеннолетние;
- наследники;
- приватизация;
- дарение;
- рента;
- перепланировка;
- зарегистрированные жильцы.

---

## 7. Правила формулировки выводов

### Можно писать
- «по доступным данным риск не выявлен»;
- «имеется вопрос, который можно закрыть документом»;
- «есть существенный риск, требующий дополнительной проверки»;
- «сделка допустима только при выполнении условий»;
- «без дополнительных документов сделка не рекомендуется».

### Нельзя писать
- «квартира абсолютно чистая»;
- «рисков нет вообще»;
- «сделка гарантированно безопасна»;
- «можно покупать без дополнительных действий».

Смысл простой: отчёт должен быть сильным, но честным. Не обещать невозможного.

---

## 8. Стоп-факторы
Если выявлено одно из нижеуказанных обстоятельств, зелёный свет на сделку давать нельзя, пока риск не снят документально или юридически.

### Безусловные стоп-факторы
- продавец не подтверждает право собственности;
- продавец не совпадает с правообладателем;
- есть арест или запрет регистрационных действий;
- есть явные признаки мошенничества;
- доверенность сомнительна или не подтверждена;
- выявлен высокий риск оспаривания права.

### Сильные стоп-сигналы
- признаки банкротства продавца;
- конфликтная наследственная история;
- проблемы с долями детей;
- неурегулированный брачный риск;
- отказ предоставить базовые документы;
- запутанная цепочка недавних перепродаж без понятной логики.

---

## 9. Что делать при нехватке данных
Если информации недостаточно, нельзя делать вид, что всё понятно.

В таком случае нужно:
1. прямо перечислить, чего не хватает;
2. указать, какие риски из-за этого нельзя закрыть;
3. дать промежуточный статус: «данных недостаточно для рекомендации по сделке».

---

## 10. Структура результата для клиента

### Короткий результат
Нужен для быстрого ответа в сообщении.

Содержит:
- уровень риска;
- 2–5 главных выводов;
- 2–5 главных рисков;
- список того, что запросить у продавца.

### Полный результат
Нужен, когда квартира реально рассматривается.

Содержит:
- карточку объекта;
- результаты проверки по разделам;
- красные флаги;
- таблицу спецрисков;
- недостающие документы;
- финальную рекомендацию.

---

## 11. Базовый сценарий общения с клиентом

### Сообщение на входе
«Пришлите адрес квартиры, кадастровый номер (если есть), ссылку на объявление и всё, что уже дал продавец: выписку ЕГРН, документы на основание права, данные продавца. Сначала сделаю экспресс-проверку и скажу, стоит ли идти дальше.»

### После экспресс-проверки
«По базовой проверке объект выглядит [спокойно / с вопросами / рискованно]. Чтобы дать нормальный вывод перед авансом, мне ещё нужны: ...»

### После полной проверки
«Я собрал риски по объекту и продавцу. Общий вывод: [уровень риска]. На сделку можно выходить только при следующих условиях: ...»

---

## 12. Внутренний стандарт качества
Отчёт считается качественным, если:
- объект точно идентифицирован;
- вывод не противоречит фактам;
- каждый серьёзный риск описан простым языком;
- указано, чем риск подтверждается;
- отдельно указано, как риск снимается;
- финальная рекомендация понятна и применима.

---

## 13. Финальный принцип
Главная задача проверки — не создать видимость уверенности, а снизить шанс плохой сделки.

Если объект хороший — это должно быть понятно.
Если объект сомнительный — это должно быть сказано прямо.
Если данных мало — это тоже нужно говорить прямо.
