# Reports / Система услуги по юридической проверке квартиры

В этой папке собран полный комплект материалов по услуге **«Юридическая проверка квартиры перед покупкой»**.

Здесь есть всё по цепочке:
- как описывать услугу;
- как принимать заявки;
- как проводить проверку;
- как оформлять выводы;
- как отдавать результат клиенту.

---

## 1. Быстрая навигация

### Главные файлы
- **`MASTER-KIT-INDEX.md`** — главный индекс по всей системе
- **`MASTER-KIT-SUMMARY.md`** — короткое summary по всему комплекту
- **`README.md`** — этот файл, быстрый гид по папке

## 2. Внутренняя методика и производство

### `apartment-legal-check-template.md`
**Что это:** полный шаблон отчёта.

**Когда использовать:**
- когда нужна глубокая проверка;
- когда объект сложный;
- когда нужен полный юридический каркас.

### `apartment-legal-check-operational-template.md`
**Что это:** боевая рабочая версия отчёта.

**Когда использовать:**
- в повседневной работе;
- для экспресс- и стандартной проверки;
- когда нужно быстро собрать практический вывод.

### `apartment-legal-check-regulation.md`
**Что это:** регламент услуги.

**Когда использовать:**
- чтобы понимать этапы;
- чтобы вести проверки единообразно;
- чтобы не пропускать стоп-факторы и ключевые риски.

## 3. Упаковка и продажа

## 6. Идеальный порядок работы: от заявки до выдачи

### Шаг 1. Получить входные данные
Использовать:
- `client-delivery-intake-form.md`
- `apartment-legal-check-launch-kit.md`

Нужно получить:
- адрес;
- кадастровый номер, если есть;
- ссылку на объявление;
- цену;
- информацию о продавце;
- всё, что уже есть из документов.

### Шаг 2. Определить формат проверки
Использовать:
- `apartment-legal-check-regulation.md`
- `apartment-legal-check-launch-kit.md`

Формат:
- экспресс;
- стандарт;
- расширенная.

### Шаг 3. Провести проверку
Использовать:
- `apartment-legal-check-operational-template.md`
- `apartment-legal-check-template.md`

Проверяются:
- объект;
- право;
- обременения;
- история переходов;
- продавец;
- специальные риски.

### Шаг 4. Сформулировать итог
Использовать:
- `client-delivery-short-result-template.md`
- `client-delivery-report-template.md`

Нужно дать:
- уровень риска;
- что подтверждено;
- какие красные флаги есть;
- чего не хватает;
- можно ли идти дальше.

### Шаг 5. Если данных не хватает
Использовать:
- `client-delivery-doc-request-template.md`

### Шаг 6. Отправить клиенту итог
Использовать:
- `client-delivery-final-message-template.md`
- `client-delivery-report-template.md`

## 7. Какую задачу решает каждый блок

### Методика
Чтобы проверка была сильной, а не поверхностной.

### Упаковка
Чтобы клиент понял ценность услуги.

### Витрина
Чтобы услугу можно было публиковать и продвигать.

### Запуск
Чтобы услугу можно было реально продавать и вести заявки.

### Клиентская выдача
Чтобы результат выглядел профессионально и понятно.

## 9. Главный принцип всей папки
Вся система строится вокруг простой идеи:

**Сначала проверить риски сделки, потом передавать деньги.**

Не делать вид, что рисков нет.
Не обещать магию.
Не заваливать клиента юридическим туманом.
А давать понятный и полезный вывод.
