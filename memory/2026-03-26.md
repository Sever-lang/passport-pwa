# 2026-03-26

- Изучены доступные скиллы и их базовые инструкции.
- Зафиксированы рабочие правила:
  - healthcheck: для аудита и харденинга хоста/OpenClaw; сначала read-only проверки, затем только по явному подтверждению любые state-changing шаги; всегда показывать нумерованные варианты; не обещать, что OpenClaw меняет firewall/SSH/OS updates сам по себе.
  - notion: работа через Notion API с обязательным `Notion-Version: 2025-09-03`; в API используются data sources; для создания страниц нужен `database_id`, для query — `data_source_id`.
  - openai-image-gen: пакетная генерация картинок через OpenAI Images API; при exec нужен повышенный timeout; есть различия параметров у gpt-image, dall-e-3 и dall-e-2.
  - openai-whisper-api: транскрибация аудио через `/v1/audio/transcriptions`; по умолчанию `whisper-1`; возможны флаги language/prompt/json/out.
  - skill-creator: скиллы должны быть компактными; SKILL.md обязателен; ресурсы делятся на scripts/references/assets; применять progressive disclosure и не плодить лишние README/CHANGELOG.
  - weather: использовать только для текущей погоды и прогноза; не для истории, тревог и спецметеоданных; основной источник — wttr.in.
  - antidetect-browser: для любых задач в браузере использовать только xvfb + Patchright через скрипты скилла; не использовать browser tool; взаимодействия только по ref из snapshot.mjs; при сбоях нужен скриншот; не завершать оплату без явного подтверждения.
  - canva: работа с Canva Connect API для designs/exports/assets/templates; нужен OAuth и env `CANVA_CLIENT_ID/SECRET`.
  - google-calendar: работа с Google Calendar API через Python-скрипт; нужны `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REFRESH_TOKEN`, `GOOGLE_CALENDAR_ID`.
  - searxng: приватный веб-поиск через локальный SearXNG; использовать `uv run .../searxng.py`; нужен `SEARXNG_URL`.
  - upx5-image-generation: при наличии присланной картинки и запросе на правку/вариацию — всегда только `edit.mjs --image`; не подменять такой запрос генерацией с нуля; генерация с нуля допустима только без входного изображения.
- Решение: эти правила считать долговременными рабочими конвенциями при выборе инструментов и скиллов.
